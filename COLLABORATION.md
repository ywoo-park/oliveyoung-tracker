# 협업 매뉴얼

올리브영 랭킹 트래커 프로젝트의 협업 가이드입니다.

---

## 역할 구분

| 역할 | 사용 도구 | 하는 일 |
|------|----------|--------|
| 비개발자 (마케터, 기획자, PM) | GitHub 웹, Slack | 이슈 작성, 결과 확인 |
| 개발자 | Claude Code, GitHub | 이슈 구현, PR 생성 |

---

## 비개발자 가이드

### 이슈 작성하기

1. **[GitHub 이슈 페이지](https://github.com/ywoo-park/oliveyoung-tracker/issues/new/choose)** 접속
2. 상황에 맞는 템플릿 선택

| 템플릿 | 언제 쓰나요? |
|--------|------------|
| 기능 요청 | 새로운 기능이 필요할 때 |
| 버그 신고 | 뭔가 잘못 동작할 때 |
| 데이터 조회 요청 | 특정 데이터나 통계가 필요할 때 |

3. 양식을 채워서 **Submit new issue** 클릭
4. Slack `#oliveyoung-tracker` 채널에 자동으로 알림이 갑니다

### 이슈 작성 팁

**좋은 이슈 예시 (기능 요청)**
```
원하는 것:
일별 순위 변화를 엑셀 파일로 다운로드하고 싶어요.

이유:
매주 월요일 팀 보고서에 첨부하는데, 지금은 화면을 보고 수동으로 옮기고 있어요.

현재 상황:
화면 캡처 후 직접 엑셀에 입력 중
```

**좋은 이슈 예시 (버그 신고)**
```
어떤 문제가 생겼나요?
어드민에서 상품을 삭제했는데 대시보드에 계속 표시됩니다.

어떻게 하다가 발생했나요?
1. 어드민 페이지 접속
2. "닥터자르트 시카페어 크림" 삭제 클릭
3. 대시보드 확인 → 아직 보임

원래 어떻게 되어야 하나요?
삭제한 상품은 대시보드에서도 사라져야 합니다.
```

### 진행 상황 확인

- 이슈 페이지에서 댓글로 진행 상황을 공유합니다
- 작업 완료 시 Slack 채널에 알림이 옵니다
- 배포 후 [대시보드](https://oliveyoung-tracker.vercel.app)에서 직접 확인 가능합니다

## 개발자 가이드

### 준비사항

- Claude Code 설치 및 실행
- GitHub MCP 연결 확인: `claude mcp list` → `github: ✓ Connected`

### 이슈 처리 흐름

#### 1. 이슈 확인

Slack 알림을 받으면 Claude Code에서 바로 조회:

```
"열린 이슈 목록 보여줘"
"이슈 #15 내용 읽어줘"
```

#### 2. 작업 시작

이슈 내용을 파악한 뒤 Claude Code에 요청:

```
"이슈 #15 기반으로 구현해줘"
"이슈 #15에서 요청한 엑셀 다운로드 기능 어디에 추가하면 좋을지 분석해줘"
```

#### 3. PR 생성

```
"작업 완료했어. 이슈 #15 기반으로 PR 만들어줘"
```

PR이 생성되면 Slack 채널에 자동 알림이 갑니다.

#### 4. 이슈에 진행 상황 코멘트 (선택)

비개발자가 현황을 알 수 있도록:

```
"이슈 #15에 '작업 시작했습니다. PR #16 참고해주세요' 코멘트 달아줘"
```

### 자주 쓰는 Claude Code 명령어

```
# 이슈 조회
"열린 이슈 목록 보여줘"
"이슈 #15 읽어줘"

# 코드 분석
"이슈 #15 구현하려면 어떤 파일을 수정해야 해?"

# PR
"PR 만들어줘, 이슈 #15 닫히도록"
"PR #16 상태 확인해줘"

# 배포 확인
"최근 커밋 목록 보여줘"
```
